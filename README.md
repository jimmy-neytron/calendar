# Workspace Calendar Vue 3 Local — v1.9.0

Компактное Vue 3 приложение-календарь с локальными аккаунтами, пространствами, календарем, спорт-вкладкой, двумя темами и уведомлениями. Supabase/backend пока не используется: все данные хранятся в `localStorage`, но структура проекта оставлена готовой к будущей миграции на backend.

## Разделы

- **Календарь** — события пространства, режимы месяц/неделя/день, повторы, фильтры, drag & drop.
- **Спорт** — программа упражнений по дням, отметки выполнения, добавление через боковую панель и JSON.
- **Настройки** — аккаунт, пространство, участники, приглашения, уведомления, внешний вид, backup.

## Новое в v1.9.0

### Улучшенные настройки

Страница настроек полностью переработана:

- вместо длинной сетки — более удобная структура с левым меню секций;
- сверху добавлен красивый summary-блок аккаунта, пространства и роли;
- блоки разделены по смыслу: аккаунт, пространство, уведомления, внешний вид, приглашения, данные, история;
- участники пространства отображаются компактнее;
- настройки лучше работают в светлой и черной теме;
- на мобильных все секции складываются в одну колонку.

### Центр уведомлений

Добавлен новый локальный центр уведомлений:

- уведомления привязаны к пользователю и активному пространству;
- если другой участник создает событие, где текущий пользователь указан участником, появляется уведомление;
- если это же событие редактируют повторно, создается не новое уведомление, а обновляется старая карточка;
- в уведомлении видно, что именно изменилось: дата, время, место, важность, заметки;
- есть счетчик обновлений одной карточки;
- есть прочитано/непрочитано;
- можно отметить все как прочитанные;
- можно очистить уведомления текущего пространства;
- уведомления хранятся в `localStorage`.

Компоненты:

```txt
src/app/components/notifications/NotificationBell.vue
src/app/components/notifications/NotificationCenter.vue
src/app/stores/notification.store.js
```

### Колокольчик в шапке

В шапку приложения добавлена кнопка уведомлений:

- показывает количество непрочитанных уведомлений;
- открывает компактный popover;
- на мобильных раскрывается как удобная панель под header.

### Логика без лишнего шума

Для каждого события и пользователя используется `dedupeKey`:

```txt
workspaceId:userId:event:eventId
```

Поэтому повторные правки одного события не создают много уведомлений. Карточка обновляется и снова становится непрочитанной.

## Основные возможности приложения

- локальные аккаунты;
- вход и регистрация без backend;
- пространства;
- приглашения по коду;
- роли в пространстве: владелец, админ, участник, просмотр;
- переключение пространства;
- календарь внутри пространства;
- режимы календаря: месяц, неделя, день;
- плавная анимация смены периода;
- drag & drop событий;
- повторы событий;
- дублирование событий;
- фильтры и поиск;
- важность события;
- напоминания;
- история действий;
- вкладка спорта;
- две темы: черная и светлая;
- экспорт/импорт JSON;
- авто-бэкап;
- проверка квоты localStorage.

## Важно

Supabase пока не подключен. Уведомления сейчас локальные: они работают между локальными аккаунтами на одном устройстве/браузере. Для настоящей синхронизации между устройствами позже нужно будет вынести аккаунты, пространства, события и уведомления в backend/Supabase.
